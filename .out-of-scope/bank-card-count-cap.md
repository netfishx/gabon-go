# 银行卡数量上限 / 列表分页

新版不设每客户银行卡数量上限，银行卡列表不做分页。

## 为什么不做

#65 cross-review 中 Claude 与 Codex 的 Standards 轴共振指出「列表无界」（偏离仓库集合分页惯例）。但一手核对旧版（Java `CustomerBankCardController`）：**旧版同样无上限、无分页**，以此形态运行多年无滥用实证。

裁决（2026-07-13，#73 triage，owner 拍板）：维持与旧版一致的无上限。理由：

- 滥用面 = 已登录客户手动添加垃圾卡，危害仅为自己名下的冗余数据行，无资损面（提现打款按显式选卡 + 管理端审核）
- 每客户实际卡数天然少（个位数），分页/上限是为不存在的规模加防御
- 加上限是产品行为变更而非质量修复，无实证支撑时不偏离 parity

若上线后出现真实滥用（监控发现异常卡量），再以运营数据为依据立票补上限——届时是一次条件查询的成本。
