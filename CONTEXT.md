# Gabon

短视频 + 邀请裂变 + 钻石钱包的社交产品后端。用户看视频、签到、做任务、邀请他人赚取钻石，钻石可购买 VIP，法币充值买钻、钻石提现变现。本仓库是旧 Java 版的 Go 全量重写。

## Language

### 账号与裂变

**客户 (Customer)**:
C 端注册用户，一切内容与资金行为的主体。
_Avoid_: 用户、User、member（与管理员混淆时尤其禁止）

**管理员 (Admin)**:
后台运营账号，与客户完全独立的身份主体，只有 ADMIN / NORMAL 两种角色。
_Avoid_: 运营、operator

**邀请码 (Invite Code)**:
每个客户唯一的 8 位注册凭码，注册时填写即建立邀请关系。

**邀请人 (Inviter)**:
邀请关系中的上级客户；奖励的接收方。
_Avoid_: 上级、推荐人

**祖先路径 (Ancestor Path)**:
一个客户在邀请树上自根到自身的完整上级链。

**团队 (Team)**:
以某客户为根、向下三级以内的全部被邀请客户。
_Avoid_: 下线、组织

**有效用户 (Valid User)**:
满足激活标准（有作品、有成功邀请、有联系方式）的客户；是触发邀请奖励的判定单位，且状态一经达成不回退。
_Avoid_: 活跃用户（那是统计口径，不是状态机）

### 钱包与支付

**钻石 (Diamond)**:
平台唯一虚拟货币，1 元 = 100 钻。
_Avoid_: 金币、积分、余额（余额指钻石的存量，不是货币本身）

**可用余额 (Available Balance)**:
客户当前可消费、可申请提现的钻石存量。

**冻结余额 (Frozen Balance)**:
提现申请提交后、审核终结前被暂扣的钻石。

**流水 (Transaction)**:
钻石余额每一次变动的不可变记录，带类型与方向。
_Avoid_: 账单、明细

**充值 (Recharge)**:
法币购入钻石。

**提现 (Withdrawal)**:
钻石变现、向客户银行卡打款；必须经人工审核。
_Avoid_: 取款、体现

**现金订单 (Cash Order)**:
一次充值或提现从发起到终态的完整生命周期记录。

**支付渠道 (Payment Provider)**:
外部三方支付服务的接入单元（pay126 / stpay / hotcl / mock）。
_Avoid_: 支付网关、通道

**代收 (Collection)**:
资金流入方向——客户付款给平台（充值）。

**代付 (Payout)**:
资金流出方向——平台向客户打款（提现）。

**取款密码 (Withdrawal Password)**:
独立于登录密码、仅用于发起提现的第二口令。

### 内容

**视频 (Video)**:
客户上传的作品，经转码与审核后方可公开。
_Avoid_: 作品、短视频（口语可用，代码与文档统一 Video）

**转码 (Transcode)**:
上传原片到 HLS 可播放格式的处理过程。

**审核 (Review)**:
人工判定内容（视频/任务证明/提现）是否放行的动作。

**Feed**:
面向所有客户的公开视频流。
_Avoid_: 推荐流、首页列表

**播放点击 (Play Click)**:
一次播放行为的发生，无时长门槛。

**有效播放 (Valid Play)**:
达到时长标准、计入热度的播放。

**热度分 (Hot Score)**:
内容互动的累计计分，只增不减（点击 +1 / 有效播放 +1 / 点赞 +2 / 评论 +5）。

**周榜 / 月榜 (Weekly / Monthly Ranking)**:
按自然周期结算的视频热度排行。

**关注 (Follow)**:
客户对客户的单向订阅；双向成立即互关 (Mutual Follow)。

### 激励

**周期任务 (Periodic Task)**:
按日/周/月自动重置、进度自动累计、达标自动发奖的任务。
_Avoid_: 日常任务

**限时领取任务 (Claim Task)**:
需客户主动领取、提交证明、经审核后发奖的限量任务。
_Avoid_: 悬赏任务

**任务证明 (Proof)**:
完成限时领取任务时提交的图片凭证（1–9 张）。

**签到 (Sign-in)**:
每日一次的打卡奖励行为，按自然月累计。

**里程碑 (Milestone)**:
当月累计签到达到指定天数时的额外奖励节点。

**VIP 等级 (VIP Level)**:
普通/铜/银/金四档付费身份，用钻石购买，只升不降、可跳级。

**倍率 (Multiplier)**:
VIP 等级对奖励金额的放大系数。

**看广告 (Ad Watch)**:
客户观看一条广告、获得奖励并消耗广告库存的行为。

**广告商 (Advertiser)**:
广告的投放主体，后台管理的对象。
