# gabon-go 全量功能清单（验收基准）

来源：2026-07-03 对旧 gabon (Java) 代码库的全量梳理。术语见 [CONTEXT.md](../CONTEXT.md)，定位见 [ADR-0002](./adr/0002-parallel-experiment.md)。

**验收规则**：清单上每一项在本项目中必须有对应实现，且至少有一条可执行的验证路径（单测 / testcontainers 集成测 / httptest E2E 之一）。API 与 schema 形状可重新设计，业务能力必须逐项对齐。

## A. 认证账号

- [ ] 注册（用户名+密码，可携带邀请码建立邀请关系）
- [ ] 登录 / 登出 / 当前用户信息 / 修改密码 / token 刷新
- [ ] JWT 鉴权，customer 与 admin 两种主体共用一套实现
- [ ] 密码哈希：argon2id（行为差异 #1）

## B. 邀请裂变

- [x] 每客户唯一 8 位邀请码；注册时写入邀请人与祖先路径（M1 落库；M4 E2E 钉死三级祖先路径语义）
- [x] 有效用户状态机：有作品 且 有成功邀请 且 有联系方式（邮箱或手机）→ 翻转为有效，不回退（M4：CAS 原子翻转，三触发点各出"最后一块拼图"E2E + 不回退断言 + 删作品不判定）
- [x] 客户翻转为有效用户时给邀请人发钻石奖励：幂等（一个被邀请人只触发一次）、受邀请人 VIP 等级的邀请奖励上限约束（M4：同事务发奖 E2E + cap 边界 + 并发集成测恰好一笔，行为差异 #11）
- [x] 团队报表：3 级以内下级列表（含头像）、团队汇总；跨级访问越权守卫（M4：下钻/越权 403/分页/汇总 E2E，行为差异 #13）

## C. 视频内容

- [x] 上传：预签名直传对象存储 → confirm 触发转码（M3：E2E 真 PUT MinIO + 归属/存在守卫）
- [x] 转码管线：DB 任务表 + 进程内 ffmpeg worker 池 → HLS + 首帧图（ADR-0003）（M3：状态机服务缝测试 + 真 ffmpeg 冒烟）
- [x] 视频状态机：待转码 → 转码中 → 待审核 → 审核通过 / 驳回（+转码失败终态）；仅审核通过进入 Feed，且触发作者 video_count+1（M3：审核 E2E 同事务断言）
- [x] Feed / 精选 / 他人主页作品 / 我的作品 / 详情 / 删除（软删）（M3：seed 稳定分页 E2E；**榜单读取页随 M7 rankings 结算落地**）
- [x] 点赞（永久落库，行为差异 #2）、每人每视频每日 1 条评论（M3：防刷分 E2E——取消再赞不计分、删评当日不可再评）
- [x] 播放上报：播放点击 与 有效播放 两种事件（M3：两段式 E2E，valid 幂等）
- [x] 标签：每视频至多 3 个（M3：confirm 守卫用例）

## D. 热度榜单

- [x] 热度计分：点击 +1 / 有效播放 +1 / 点赞 +2 / 评论 +5，只增不减（M3：互动 E2E 逐权重断言）
- [ ] 累计热度 + 周榜/月榜；cron 按自然周期结算（周一 00:01 / 每月 1 日 00:02，Asia/Shanghai）

## E. 关注

- [ ] 关注 / 取关（软删复用记录）；关注状态三态：未关注 / 已关注 / 互关
- [ ] 粉丝数、关注数统计（不复刻假数据 hack，行为差异 #4）

## F. 钱包交易

- [x] 可用余额 + 冻结余额；1 元 = 100 钻（M2：查询端点 E2E + 冻结不变量集成测）
- [x] 全部余额变更走原子条件 UPDATE（扣减需 `WHERE balance >= amount`），杜绝超扣（M2：并发扣减集成测 + property 对账恒等式）
- [x] 流水记录：类型覆盖 提现 / 充值 / 观看奖励 / 任务奖励 / 签到奖励 / 邀请有效奖励 / 内容奖励 / VIP 购买（类型拆细为 10 类统一一套，行为差异 #6；落笔机制 M2 完成，各奖励触发链路随所属域里程碑接入）

## G. 充值提现

- [ ] 充值：创建现金订单 → 调支付渠道代收 → 免鉴权回调（验签+幂等）→ 到账
- [ ] 提现：校验取款密码 → 冻结钻石 → 待审核；审核通过走代付、驳回解冻
- [ ] 提现审核由 admin 路由进程内调用完成（行为差异 #5），订单号确定性生成保证幂等
- [ ] 回调幂等三重保障：订单号唯一 + (渠道, 渠道订单号) 唯一 + 已终态短路
- [ ] 支付渠道：Provider 接口 + 注册表；移植 pay126（JSON/MD5 小写/收付双商户）、stpay（GET 回调/分单位/MD5 大写）、hotcl（form/分/MD5 大写）三渠道实现但默认不启用；内置 mock provider 支撑本地 E2E 全链路
- [ ] 银行卡 CRUD（提现收款用）
- [ ] 充值超时取消 cron：超时订单先向渠道查账再取消

## H. VIP

- [ ] 四级（普通/铜/银/金），钻石全价购买、仅可升级、可跳级、CAS 防并发重复购买
- [ ] 等级配置表驱动（价格、奖励倍率、邀请奖励上限）

## I. 任务系统

- [ ] 限时领取任务：VIP 门槛 → 领取 → 提交证明（1–9 张图，须本人上传的存储路径）→ 审核 → 发奖；六态状态机；过期 cron（每 5 分钟）

## J. 签到活动

## 行为差异（明确不复刻旧行为）

1. **密码算法**：argon2id，不复刻自制 `salt$sha256`（无数据迁移，无兼容义务）
2. **点赞**：永久落库。旧版只存 Redis 且 7 天 TTL 过期即"未赞"、like_count 漂移，是缺陷
3. **Feed 分页**：弃 `ORDER BY RAND()`（翻页重复/漏），改 seed 随机或热度加权
4. **假粉丝数 hack**：旧版对特定 id 区段返回随机粉丝数，砍掉
5. **admin→service 内部 HTTP**（提现审核 + X-Internal-Secret）：单二进制内进程调用，接口消失
6. **错误码**：不复刻 114 个 BizCode（含重复码）与两套互相矛盾的订单类型码，统一重新设计
7. **JWT**：旧版两套实现（过期时间单位都不一致），合并为一套
8. **手动补领奖励**：旧版"读状态再写"有并发双发风险，统一条件 UPDATE
9. **保留**：软删语义、Asia/Shanghai 时区锚点；分发改走阿里云 CDN 回源 S3（ADR-0005），防盗链维持"不签名"现状（如启用，用阿里云 CDN URL 鉴权，不移植旧 CdnSignUtil）
10. **不移植僵尸依赖**：Redisson、xxl-job、kaptcha 在旧版引而未用
11. **邀请奖励发放**：纯配置驱动（配置缺行/停用即不发），废除旧版 123 钻代码硬编码兜底（种子行以 123 对齐现网实跑值）；cap 检查对邀请人行加锁串行化，不复刻旧版并发超发竞态（M4）
12. **手机号正则**：`^1[3-9]\d{9}$`，补全旧版遗漏的 16x/19x 号段（M4）
13. **团队越权**：明确 403 错误码，不复刻旧版静默返回空列表；下钻边界按团队定义（3 级以内）放宽到深度 2 的 parent，深度 3 成员的下级计数归 0 不泄漏界外结构（M4）
14. **联系方式唯一性**：phone/email 数据库部分唯一索引兜底，不复刻旧版仅应用层 count 检查的并发竞态（M4）
15. **删除已发布视频回退 `video_count`**：旧版不减计数，删光作品后仍可被判定"有作品"翻转有效并发奖，是缺陷不复刻（M4）
16. **签到舍入与流水拆分**：日签与里程碑各自 `floor(base×倍率bp/10000)`、各自入账两笔（`sign_in_reward`/`milestone_reward`），不复刻旧版"合并基数后 HALF_DOWN 单笔"，结果可差 ±1 钻（M5）

## 技术决策附录

| 决策点 | 结论 | 备注 |
|--------|------|------|
| 语言 | Go | ADR-0001 |
| 定位 | 平行实验品 | ADR-0002 |
| 形态 | 单 module、单二进制、单服务（/api + /admin 同进程） | |
| HTTP | chi | handler 全程标准库签名 |
| 数据访问 | sqlc + pgx | SQL-first，编译期类型安全 |
| 数据库 | PostgreSQL（唯一数据层） | ADR-0004 |
| 迁移 | goose | 纯 SQL 迁移文件 |
| 定时任务 | 进程内 cron (robfig/cron) | 旧版 8 个 @Scheduled 对应移植 |
| 转码 | ffmpeg + DB 队列 + worker 池 | ADR-0003 |
| 对象存储 | S3 协议、endpoint 可配 | 生产 AWS S3 / 本地 MinIO |
| CDN | 阿里云 CDN，回源 S3 | 播放 URL 走 CDN 域名，域名可配 |
| 云服务约束 | 仅 S3 与阿里云 CDN 豁免，其余自托管/进程内 | ADR-0005 |
| 支付 | Provider 接口 + 三真实渠道（默认停用）+ mock | |
| 日志 | slog 结构化 JSON | |
| 配置 | 环境变量 → 单 config struct，启动时校验 fail fast | 密钥不入文件 |
| API 文档 | 不引 swagger 生成器 | |
| 测试 | 三层：table-driven 单测 / testcontainers 真库集成 / httptest E2E | ffmpeg 用小样本真转码冒烟 |
